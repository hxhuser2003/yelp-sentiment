## Project Overview

This project addresses fine-grained sentiment analysis: predicting a 1–5 star rating from raw Yelp review text. We systematically compare four approaches:

1. **Logistic Regression** with TF-IDF + GloVe + handcrafted features (ablation study)
2. **Linear SVM** with the same feature combinations
3. **RoBERTa** full fine-tuning with MPS acceleration (Apple M2)
4. **GPT-4o-mini** prompting with four strategies (zero-shot, few-shot, CoT, CoT+few-shot)

Beyond standard accuracy metrics, we implement **LLM-as-Judge** to characterize the qualitative nature of each model's errors.

## Data Setup

### Step 1: Download and split the dataset

```bash
python scripts/load_data.py
```

This downloads Yelp Review Full from HuggingFace, draws a stratified subsample of 50,000 reviews, and saves train/val/test splits to `data/processed/`.

### Step 2: Download GloVe embeddings (optional, needed for GloVe ablation)

```bash
mkdir -p data/raw
curl -L https://nlp.stanford.edu/data/glove.6B.zip -o data/raw/glove.6B.zip
unzip data/raw/glove.6B.zip glove.6B.100d.txt -d data/raw/
```

## Running the Pipeline

### Step 2: Train Logistic Regression

```bash
python scripts/train_lr.py
```

- Runs feature ablation (TF-IDF / TF-IDF+GloVe / TF-IDF+GloVe+HC)
- Grid search over C ∈ {0.01, 0.1, 1, 10} on validation set
- Saves best model to `models/lr_tfidf_only.pkl`
- Results saved to `results/lr_ablation.json`

### Step 3: Train Linear SVM

```bash
python scripts/train_svm.py
```

- Same ablation and grid search as LR
- Saves best model to `models/svm_tfidf_only.pkl`
- Results saved to `results/svm_ablation.json`

### Step 4: Fine-tune RoBERTa

```bash
# Prevent sleep on Mac during long training
caffeinate -i python scripts/train_roberta.py 2>&1 | tee results/roberta_log.txt
```

- Fine-tunes `roberta-base` for 5-class classification
- Uses MPS acceleration on Apple M2
- Training takes ~60 minutes for a single config (lr=2e-5, bs=16)
- Best model saved to `models/roberta/lr2e-05_bs16_best.pt`
- Results saved to `results/roberta_results.json`

**Note:** If you encounter memory issues, close other applications (Chrome, etc.) before running.

### Step 5: GPT-4o-mini Prompting

```bash
# Set your OpenAI API key first
export OPENAI_API_KEY="sk-your-key-here"

python scripts/gpt_prompting.py
```

- Evaluates 4 prompt strategies on 50 test samples each
- Requires OpenAI API key with sufficient credits (~$0.01)
- Rate-limited to 3 RPM for new accounts (~70 minutes total)
- Results saved to `results/gpt_*.json`

### Step 6: LLM-as-Judge Error Analysis

```bash
export OPENAI_API_KEY="sk-your-key-here"

python scripts/llm_judge.py
```

- Samples 50 errors per model and asks GPT-4o-mini to classify each as reasonable or severe
- Requires Steps 2–5 to be completed first
- Results saved to `results/judge_*.json`

### Step 7: Generate Figures

```bash
python scripts/visualize_dataset.py    # Dataset figures
python scripts/visualize_loss_curves.py  # Hyperparameter tuning curves
python scripts/visualize.py            # All result figures
```

All figures saved to `results/figures/`.

## Notes on Reproducibility

- All random seeds are fixed to **42** throughout
- TF-IDF and StandardScaler are fit on training set only
- RoBERTa tokenizer uses `padding=max_length` with `max_length=128`
- GPT experiments use `temperature=0` for deterministic outputs
- Stratified sampling ensures identical class distributions across all runs
